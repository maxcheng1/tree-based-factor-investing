# Tree-Based Machine Learning for Factor Investing

Course project for ACTSC 423 / AFM 423 (University of Waterloo). The project tests whether tree-based models (random forest, XGBoost) improve on linear benchmarks for US equity factor investing. Stocks are labeled by next-month return quintile (strong sell → strong buy), predicted class probabilities drive position sizing, and all models are evaluated out of sample with rolling walk-forward validation.

## Repository structure

```
tree-based-factor-investing/
├── data/                # Local data storage; contents are gitignored (licensed WRDS data)
│   ├── raw/             # Unmodified extracts from WRDS and the Kenneth French Data Library
│   └── processed/       # Cleaned, merged, model-ready panels built by the pipeline
├── src/                 # Importable Python modules: data wrangling, features, models, validation
├── scripts/             # Command-line entry points that run the pipeline end to end
├── notebooks/           # Jupyter notebooks for exploration and prototyping
├── tests/               # Unit tests for src/ modules
├── results/
│   ├── figures/         # Generated charts (.png) used in the report
│   └── tables/          # Generated performance tables (.csv)
├── requirements.txt     # Pinned Python dependencies
├── LICENSE
└── README.md
```

### Conventions

- **`data/` contents stay local.** CRSP and Compustat data are licensed through WRDS and cannot be redistributed, so every file under `data/` is gitignored. The `.gitkeep` placeholders preserve the folder layout for anyone cloning the repo.
- **`src/` holds logic; `scripts/` holds entry points.** Modules in `src/` define reusable, documented functions with all hyperparameters, file paths, and random seeds passed as arguments. Scripts import those modules and execute the full workflow.
- **`notebooks/` is exploratory.** Any code needed for final results gets promoted into `src/`, so the complete pipeline runs from the command line.
- **`results/` holds generated artifacts only.** Every figure and table can be regenerated by re-running the pipeline. Figures export as `.png`.

## Data access

This project uses licensed data that is excluded from the repository. Researchers with WRDS access can rebuild the dataset from the following sources (exact table names, variable lists, and filters are documented alongside the wrangling code as the pipeline is finalized):

- **CRSP monthly stock file** — monthly returns, prices, and shares outstanding, approx. 1990–2025
- **Compustat fundamentals** — accounting variables for factor construction, lagged to prevent look-ahead bias
- **CRSP/Compustat Merged (CCM) link table** — links the two databases
- **Fama-French 5 factors** — from the Kenneth French Data Library (publicly available)

## Setup

Option A — conda:

```bash
conda create -n gpr python=3.11 -y
conda activate gpr
pip install -r requirements.txt
```

Option B — pip/venv:

```bash
python -m venv .venv
source .venv/Scripts/activate    # Windows Git Bash; use .venv/bin/activate on macOS/Linux
pip install -r requirements.txt
```

## Reproducing results

```bash
python scripts/run_pipeline.py
```

(Entry point under development.) All stochastic steps — sampling, model fitting, and validation splits — take explicit seed arguments, and the default values reproduce the results reported in the paper.
